Pipeline 1 · TVM Data Ingest
Daily Logic App pulls vulnerability data from MDE API into a Sentinel custom table. Independent component, no other modules required.

⚙️ Logic App

Scheduled daily. Uses managed identity to call SoftwareVulnerabilitiesByMachine endpoint and writes results to MDETVM_CL.

CommercialGCC/GCCH
  • MDETVM-LogicApp.json
  • MDETVM-LogicApp.gov.json

🔑 Permissions

Grants Vulnerability.Read.All on the Windows Defender ATP app role to the Logic App's managed identity.

  • Assign-MDVMPermissions.ps1

✅ Verification & Health

KQL queries to confirm MDETVM_CL table exists and has recent data. Alert rules for pipeline failures.

  • Verify-MDVMTables.kql
  • Pipeline-Health-Alerts.json
▼
Pipeline 2 · KEV Detection
Scheduled analytics rule correlates MDETVM_CL against CISA's live KEV catalog using externaldata. Creates Sentinel incidents for active KEVs.

🔍 Analytics Rule

KQL joins MDETVM_CL with CISA KEV CSV via externaldata(). Filters out watchlist exceptions. Maps entities for incident graph.

KQL
  • CISA-KEV-MDVM-Correlation.kql
  • CISA-KEV-MDVM-AnalyticsRule.json

📋 Exceptions Watchlist

CSV-based watchlist for approved exceptions. Excludes specific CVE+device combinations from triggering incidents.

  • KEV-Exceptions-Watchlist.json
▼
Pipeline 3 · Automated Remediation
Automation rule triggers a Logic App playbook on each KEV incident. Routes to WUfB (Windows KBs) or Intune (third-party apps). Verifies and auto-closes.

🤖 Remediation Logic App

Parses incident CVE/KB data. For Windows CVEs with KBs: creates WUfB expedited update deployment. For third-party: triggers Intune remediation.

  • KEV-Remediate-LogicApp.json
  • KEV-Remediate-AutomationRule.json

🪟 WUfB Path

Enrolls devices via Graph enrollAssetsById, then adds to deployment audience. Expedites the specific security KB.

PowerShell
  • Assign-KEVRemediatePermissions.ps1
  • Setup-IntuneBaseline.ps1

📦 Third-Party Path

For apps without KBs (7-Zip, Notepad++, etc.). Routes through Intune remediation scripts or Teams approval workflow.

  • Patch-ThirdPartyApprovalPath.ps1
  • Update-KEVRemediateThirdPartyPath.ps1
  • Intune-KEV-Starter-Policy.md

🔄 Auto-Close & Verify

Re-queries MDETVM_CL after remediation. If CVE count drops to zero for the device, closes the Sentinel incident with proof.

CommercialGCC/GCCH
  • AutoClose-KEVIncidents-LogicApp.json
  • AutoClose-KEVIncidents-LogicApp.gov.json
  • Verify-Remediation.ps1

Interactive dashboards for KEV exposure tracking, device vulnerability heatmaps, and remediation progress. Requires only TVM Data Ingest.

Key Lessons & Gotchas

  • WinGet does NOT work as SYSTEM. Use registry version checks for detection + direct download/silent install for remediation.
  • WUfB requires both enrollAssetsById (quality) AND audience add. Audience alone leaves devices as notEnrolled.
  • Logic Apps make_set() returns a JSON string, not an array. Parse with json(coalesce(...)).
  • Graph $top caps at 5 for Windows Update catalog entries. Use $expand=productRevisions to map KBs.
  • After triggering MDETVM ingest, wait before querying. Indexing lag can cause false "still vulnerable" reads.
  • Gov environments: use api-gcc / api-gov endpoints and .gov.json Logic App variants.
